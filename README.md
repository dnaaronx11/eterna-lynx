# EternaLynX
Project Overview
EternaLynX is a next-generation Network Operating System (NetOS) that unifies social interaction, commerce, finance, creative expression, education, and holographic spatial computing into a single, sovereign digital ecosystem. Unlike traditional siloed platforms, EternaLynX operates as a unified security perimeter with six logically isolated planes connected by secure, consent-enforced bridges.

Vision
Market Dominance Before 2027: Become the default digital interaction layer globally
User Sovereignty: Explicit, revocable consent over all data movement
Post-Quantum Security: Cryptographic agility to survive future computing paradigms
AI-Augmented Autonomy: Autonomous agents (PersonaLynX, Athena, DigiFriend) that enhance rather than replace human judgment
Core Differentiators
✅ Twin-Blockchain Architecture (Guardian Chain for security, Velocity Chain for transactions)
✅ Steganographic Key Management (encryption keys hidden in user-selected images)
✅ DAG-Enhanced Orchestration (parallel processing for high throughput)
✅ Hierarchical Multi-Agent System (HMAS) (autonomous agents coordinating across planes)
✅ Trusted Execution Environments (TEEs) (hardware-enforced isolation for critical workloads)
✅ Model Context Protocol (MCP) (standardized AI agent tool integration with delegated signing)

Quick Start (Phase 9)
Prerequisites
Windows 11 (or Linux/macOS with Docker Desktop)
Docker Desktop installed and running
Python 3.13 (optional, for AI services)
Node.js 18+ (for backend services)
Step 1: Clone the Repository
bash
git clone https://github.com/YourOrg/EternaLynX.git
cd EternaLynX
Step 2: Configure Environment Variables
bash
# Copy the example environment file
cp .env.example .env

# Edit .env with your secrets (DO NOT COMMIT THIS FILE)
# Required variables:
# - POSTGRES_PASSWORD (complex password)
# - OPENAI_API_KEY (for Quantum Nexus AI inference)
# - JWT_SECRET_KEY (for session tokens)
Step 3: Launch the Entire Ecosystem
bash
# Build all services and start the containerized network
docker compose up --build -d

# Verify services are running
docker compose ps

# View logs for debugging
docker compose logs -f api_gateway
Step 4: Access the System
EternaNet Browser Shell: http://localhost:3003
API Gateway Dashboard: http://localhost:8080/dashboard
Traefik Admin Panel: http://localhost:8080
PostgreSQL (Internal): localhost:5432 (use psql CLI)
Redis (Internal): localhost:6379 (use redis-cli)
Step 5: Test a Transaction
bash
# Example:  Create a user in SociaLynX
curl -X POST http://localhost:8080/social/v1/profile/create \
  -H "Content-Type: application/json" \
  -d '{"username": "test_user", "email": "test@example.com"}'

Troubleshooting
Service Won't Start:

bash
# Check logs for errors
docker compose logs service_name

# Rebuild from scratch
docker compose down -v
docker compose up --build
Database Migration Issues:

bash
# Apply pending migrations
docker exec backend_service npm run migrate: latest

# Rollback if needed
docker exec backend_service npm run migrate:rollback
Out of Memory:

bash
# Increase Docker Desktop allocation
# Settings > Resources > Memory:  increase to 32+ GB

docker stats
